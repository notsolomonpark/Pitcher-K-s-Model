## üöÄ Quick Start - Deploy to GitHub Pages

### Step 1: Create GitHub Repository

1. Go to [github.com](https://github.com) and sign in
2. Click the **"+"** button (top right) ‚Üí **"New repository"**
3. Repository name: `strikeout-predictions` (or any name you prefer)
4. Make it **Public** (required for free GitHub Pages)
5. Click **"Create repository"**

### Step 2: Upload Files

#### Option A: Via Web Interface (Easiest)
1. In your new repository, click **"uploading an existing file"**
2. Drag and drop these files:
   - `index.html`
   - `style.css`
   - `app.js`
   - `predictions.json`
3. Click **"Commit changes"**

#### Option B: Via Git Command Line
```bash
# Navigate to this folder
cd github-pages

# Initialize git
git init
git add .
git commit -m "Initial commit"

# Add remote (replace YOUR-USERNAME)
git remote add origin https://github.com/YOUR-USERNAME/strikeout-predictions.git

# Push to GitHub
git branch -M main
git push -u origin main
```

### Step 3: Enable GitHub Pages

1. In your repository, click **"Settings"** (top menu)
2. Click **"Pages"** (left sidebar)
3. Under **"Source"**, select:
   - Branch: `main`
   - Folder: `/ (root)`
4. Click **"Save"**
5. Wait 2-3 minutes for deployment

### Step 4: Access Your Site

Your site will be live at:
```
https://YOUR-USERNAME.github.io/strikeout-predictions/
```

## üîß Troubleshooting

### Site not loading?
- Check that GitHub Pages is enabled in Settings ‚Üí Pages
- Make sure repository is Public
- Wait 2-3 minutes after first deployment

### Predictions not showing?
- Check browser console for errors (F12)
- Verify `predictions.json` is valid JSON format
- Make sure file is committed to the repository

### Old data showing?
- Hard refresh: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
- Clear browser cache
- GitHub Pages can take 1-2 minutes to update
